# GMC Feed for WooCommerce

WordPress плагін для генерації фідів товарів Google Merchant Center з підтримкою WooCommerce та WPML.

## Опис

GMC Feed for WooCommerce (раніше D14K Merchant Feed) автоматично генерує XML-фіди товарів для Google Merchant Center з вашого інтернет-магазину WooCommerce. Плагін повністю підтримує багатомовність через WPML та автоматично створює окремі фіди для кожної активної мови.

**Важливо:** Плагін містить захист від використання на сайтах у доменній зоні `.ru`.

## Основні можливості

- ✅ **Автоматична генерація фідів** - створення XML-фідів за розкладом (кожні 3, 6, 12 годин, щодня, щотижня або щомісяця)
- ✅ **Підтримка WPML** - окремі фіди для кожної мови з автоматичним перекладом
- ✅ **Варіативні товари** - підтримка всіх варіацій товарів WooCommerce
- ✅ **Валідація товарів** - автоматична перевірка відповідності вимогам Google Merchant Center
- ✅ **Маппінг категорій** - призначення Google Product Categories для ваших категорій WooCommerce
- ✅ **Зображення з фолбеком** - автоматичне використання зображень з оригінальної мови, якщо вони відсутні в перекладі
- ✅ **Статистика генерації** - відстеження кількості товарів, помилок та пропущених позицій
- ✅ **Ручна генерація** - можливість згенерувати фіди вручну в будь-який момент
- ✅ **Гео-обмеження** - автоматичне блокування роботи на доменах окупанта (.ru)

## Вимоги

- WordPress 5.8 або вище
- WooCommerce (будь-яка актуальна версія)
- PHP 7.4 або вище
- WPML (опціонально, для багатомовності)

## Встановлення

1. Завантажте ZIP-файл плагіна
2. Перейдіть до WordPress Admin → Плагіни → Додати новий
3. Натисніть "Завантажити плагін" та виберіть ZIP-файл
4. Натисніть "Встановити зараз"
5. Активуйте плагін після встановлення

## Налаштування

1. Перейдіть до **WooCommerce → GMC Feed** в адмін-панелі WordPress
2. Налаштуйте основні параметри:
   - **Увімкнути фід** - активація автоматичної генерації
   - **Інтервал оновлення** - як часто генерувати фіди (3, 6, 12 годин, щодня, щотижня або щомісяця)
   - **Бренд** - назва вашого бренду (за замовчуванням: "Diamonds 14K")
   - **Google Product Category** - категорія за замовчуванням для всіх товарів

3. **Маппінг категорій Google**:
   - Призначте відповідні Google Product Categories для кожної категорії WooCommerce
   - Якщо категорія не вказана, використовується категорія за замовчуванням
   - Приклад: `Apparel & Accessories > Jewelry > Rings`

4. Натисніть **"Зберегти налаштування"**

## Використання

### Автоматична генерація

Після активації плагіна та налаштування, фіди будуть автоматично генеруватися за вказаним розкладом. Ви можете перевірити час наступної генерації на сторінці налаштувань.

### Ручна генерація

Натисніть кнопку **"Згенерувати зараз"** на сторінці налаштувань, щоб негайно створити фіди для всіх активних мов.

### URL фідів

Після генерації фіди доступні за наступними URL:

```
https://your-site.com/merchant-feed/uk/
https://your-site.com/merchant-feed/ru/
https://your-site.com/merchant-feed/en/
```

Де `uk`, `ru`, `en` - коди активних мов у вашому WPML.

### Додавання фіду в Google Merchant Center

1. Скопіюйте URL фіду для потрібної мови (кнопка "Копіювати" на сторінці налаштувань)
2. Перейдіть до Google Merchant Center
3. Виберіть **Products → Feeds**
4. Натисніть **"+"** щоб додати новий фід
5. Виберіть країну та мову
6. Вставте URL фіду
7. Налаштуйте розклад оновлення (рекомендується: щодня)

## Інтеграція з WPML

Якщо WPML встановлено та активовано:

- Плагін автоматично визначає всі активні мови
- Для кожної мови генерується окремий фід
- Товари автоматично перемикаються на відповідну мову
- Якщо зображення відсутнє в перекладі, використовується зображення з оригінальної мови
- URL товарів автоматично адаптуються під мову

Якщо WPML не встановлено, плагін працює з основною мовою WordPress.

## Валідація товарів

Плагін автоматично перевіряє кожен товар на відповідність вимогам Google Merchant Center:

- ✅ Наявність назви товару
- ✅ Ціна більше 0
- ✅ Наявність зображення
- ✅ Валідний URL товару

Товари, що не пройшли валідацію, пропускаються та відображаються в статистиці як "пропущено".

## Структура фіду

Кожен товар у фіді містить:

- `id` - ID варіації товару
- `item_group_id` - ID батьківського товару (для групування варіацій)
- `title` - назва товару
- `description` - опис товару
- `link` - URL товару
- `image_link` - основне зображення
- `additional_image_link` - додаткові зображення (до 10)
- `price` - звичайна ціна
- `sale_price` - ціна зі знижкою (якщо є)
- `sale_price_effective_date` - період дії знижки
- `availability` - наявність (in stock / out of stock / preorder)
- `condition` - стан (завжди "new")
- `brand` - бренд
- `google_product_category` - категорія Google
- `product_type` - категорії WooCommerce
- `color` - колір (атрибут `pa_color`)
- `material` - матеріал (атрибут `pa_gold-standard`)
- `identifier_exists` - чи є ідентифікатори (завжди "no")
- `is_bundle` - чи є набором (завжди "no")
- `shipping` - вартість доставки (0)

## Атрибути товарів

Плагін автоматично використовує наступні атрибути WooCommerce:

- `pa_color` → `g:color` (колір товару)
- `pa_gold-standard` → `g:material` (матеріал, наприклад, проба золота)

Ви можете додати ці атрибути до своїх товарів для більш детального опису в Google Merchant Center.

## Розташування файлів

Згенеровані XML-фіди зберігаються в:

```
/wp-content/uploads/d14k-feeds/feed-{lang}.xml
```

Де `{lang}` - код мови (uk, ru, en тощо).

## Усунення несправностей

### Фід не генерується

1. Перевірте, чи активовано опцію "Увімкнути фід"
2. Перевірте логи WordPress на наявність помилок
3. Переконайтеся, що WooCommerce активовано
4. Перевірте права доступу до директорії `/wp-content/uploads/`

### Товари пропускаються

Перевірте статистику на сторінці налаштувань - там відображається кількість пропущених товарів. Основні причини:

- Відсутня ціна або ціна = 0
- Відсутнє зображення
- Порожня назва товару
- Невалідний URL

### Помилки в Google Merchant Center

1. Переконайтеся, що всі обов'язкові поля заповнені
2. Перевірте, чи правильно вказані Google Product Categories
3. Переконайтеся, що зображення доступні (не захищені паролем)
4. Перевірте, чи URL товарів доступні публічно

### Cron не працює

Якщо автоматична генерація не відбувається:

1. Перевірте, чи працює WordPress Cron на вашому сервері
2. Деякі хостинги вимагають налаштування системного cron
3. Використовуйте плагін типу "WP Crontrol" для діагностики
4. Як альтернатива, використовуйте ручну генерацію
