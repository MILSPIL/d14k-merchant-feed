# GMC Feed for WooCommerce

WordPress плагін для генерації фідів товарів Google Merchant Center з підтримкою WooCommerce та WPML.

## Опис

GMC Feed for WooCommerce автоматично генерує XML-фіди товарів для Google Merchant Center з вашого інтернет-магазину WooCommerce. Плагін підтримує багатомовність через WPML та автоматично створює окремі фіди для кожної активної мови.

**Важливо:** Плагін містить захист від використання на сайтах у доменній зоні `.ru`.

## Основні можливості

- Автоматична генерація XML-фідів за розкладом (3, 6, 12 годин, щодня, щотижня або щомісяця)
- Підтримка WPML з окремими фідами для кожної мови
- Справжня мультивалютність через WCML з автоматичним підбором валюти для кожної мови
- Варіативні товари: підтримка всіх варіацій WooCommerce
- Валідація товарів за вимогами Google Merchant Center
- Маппінг категорій WooCommerce до Google Product Categories
- Custom Labels 0-4 з маппінгом по категоріях та перовизначенням на рівні товару
- Фільтрація за атрибутами: виключення товарів за значеннями атрибутів
- Кастомні правила фільтрації товарів
- Зображення з фолбеком на оригінальну мову, якщо переклад відсутній
- Статистика генерації з відображенням кількості товарів, помилок і пропущених позицій
- Ручна генерація в будь-який момент
- Гео-обмеження: блокування роботи на доменах .ru

## Вимоги

- WordPress 5.8 або вище
- WooCommerce (будь-яка актуальна версія)
- PHP 7.4 або вище
- WPML (опціонально, для багатомовності)

## Встановлення

1. Завантажте ZIP-файл плагіна
2. Перейдіть до WordPress Admin, потім Плагіни, потім Додати новий
3. Натисніть "Завантажити плагін" та виберіть ZIP-файл
4. Натисніть "Встановити зараз"
5. Активуйте плагін після встановлення

## Налаштування

1. Перейдіть до **WooCommerce → GMC Feed** в адмін-панелі WordPress
2. Налаштуйте основні параметри:
   - **Увімкнути фід** - активація автоматичної генерації
   - **Інтервал оновлення** - як часто генерувати фіди
   - **Час генерації** - конкретний час доби з урахуванням часового поясу сервера
   - **Бренд** - назва вашого бренду або атрибут WooCommerce для автоматичного підбору
   - **Google Product Category** - категорія за замовчуванням для всіх товарів

3. **Маппінг категорій Google:**
   - Призначте відповідні Google Product Categories для кожної категорії WooCommerce
   - Якщо категорія не вказана, використовується категорія за замовчуванням
   - Приклад: `Apparel & Accessories > Jewelry > Rings`

4. **Custom Labels (0-4):**
   - Для кожного лейбла налаштуйте маппінг: категорія WooCommerce → значення лейбла
   - Перовизначення на рівні товару через поле `_d14k_custom_label_X` у метаданих

5. **Фільтрація:**
   - Виключення цілих категорій (автоматично поширюється на всі переклади WPML)
   - Виключення за значеннями атрибутів
   - Кастомні правила

6. Натисніть **"Зберегти налаштування"**

## Використання

### Автоматична генерація

Після активації та налаштування фіди генеруються за вказаним розкладом. Час наступної генерації відображається на сторінці налаштувань.

### Ручна генерація

Натисніть кнопку **"Згенерувати зараз"** на сторінці налаштувань для негайного створення фідів для всіх активних мов.

### URL фідів

```
https://your-site.com/merchant-feed/uk/
https://your-site.com/merchant-feed/en/
```

Де `uk`, `en` — коди активних мов у WPML.

### Додавання фіду в Google Merchant Center

1. Скопіюйте URL фіду (кнопка "Копіювати" на сторінці налаштувань)
2. Перейдіть до Google Merchant Center, Products, Feeds
3. Натисніть "+" для нового фіду
4. Оберіть країну та мову
5. Вставте URL фіду

## Інтеграція з WPML

Якщо WPML встановлено та активовано:

- Плагін автоматично визначає всі активні мови
- Для кожної мови генерується окремий фід
- **Мультивалютність:** ціна конвертується згідно з налаштувань WCML. Якщо базова ціна товару $10, а курс 1 USD = 43.5 UAH, то у фід для української мови потрапить 435 UAH.
- При виключенні категорій автоматично виключаються й усі їхні переклади
- Товари автоматично перемикаються на відповідну мову
- URL товарів адаптуються під кожну мову

Якщо WPML не встановлено, плагін працює з основною мовою WordPress.

## Структура фіду

Кожен товар у фіді містить:

- `id` — ID варіації товару
- `item_group_id` — ID батьківського товару
- `title` — назва товару
- `description` — опис
- `link` — URL товару
- `image_link` — основне зображення
- `additional_image_link` — додаткові зображення (до 10)
- `price` — звичайна ціна
- `sale_price` — ціна зі знижкою (якщо є)
- `sale_price_effective_date` — період дії знижки
- `availability` — наявність (in stock / out of stock / preorder)
- `condition` — стан (new)
- `brand` — бренд
- `google_product_category` — категорія Google
- `product_type` — категорії WooCommerce
- `color` — колір (атрибут `pa_color`)
- `material` — матеріал (атрибут `pa_gold-standard`)
- `custom_label_0` ... `custom_label_4` — кастомні лейбли для Google Ads
- `country_of_origin` — країна виробника (якщо вказано)
- `shipping` — вартість доставки
- `identifier_exists` — no

## Усунення несправностей

### Фід не генерується

1. Перевірте, чи активовано опцію "Увімкнути фід"
2. Перевірте логи WordPress на наявність помилок
3. Переконайтеся, що WooCommerce активовано
4. Перевірте права доступу до `/wp-content/uploads/`

### Товари пропускаються

Основні причини: відсутня ціна або ціна 0, відсутнє зображення, порожня назва, невалідний URL. Статистика відображається на сторінці налаштувань.

### Cron не працює

1. Перевірте, чи працює WordPress Cron на вашому сервері
2. Деякі хостинги вимагають налаштування системного cron замість WP-Cron
3. Використовуйте плагін WP Crontrol для діагностики

## Changelog

### 2.1.0

**Адмін-інтерфейс**

Повний редизайн сторінки налаштувань у SaaS-стилі. Замість стандартного WordPress-вигляду тепер використовується власна система дизайну на базі CSS-змінних. Темна шапка з логотипом та версією плагіна, навігація по вкладках з іконками, картки з рівними відступами 16px з усіх сторін. Вкладка "Довідка" зі списком поширених запитань, контактним банером і виправленими SVG-іконками знаку питання (крапка тепер видима).

**Custom Labels**

Додано повноцінну підтримку Custom Labels 0-4 для Google Ads. Для кожного лейбла налаштовується маппінг: категорія WooCommerce → значення лейбла. Якщо один товар потрапляє одразу в кілька категорій, спрацьовує перша знайдена. Перовизначення на рівні конкретного товару через мета-поле `_d14k_custom_label_X` (вищий пріоритет над маппінгом). Реалізовано у новому класі `class-product-meta.php`.

**Фільтрація товарів**

Три рівні фільтрації: виключення категорій цілком (з автоматичним поширенням на переклади WPML), виключення за значеннями атрибутів і кастомні правила. При виключенні варіативного товару за категорією виключаються всі його варіації.

**Планувальник cron**

Новий метод `reschedule_at_time()` дозволяє встановити конкретний час генерації з урахуванням часового поясу. Якщо вказаний час на сьогодні вже минув, генерація переноситься на завтра. Підтримка стандартних PHP-таймзон.

**WPML**

При виключенні категорій список автоматично розширюється на всі WPML-переклади кожного терміна. Custom Labels коректно резолвляться через оригінальний term_id для мов, де категорії є перекладами.

**XML-фід**

Додано підтримку тегів `g:country_of_origin` та `g:shipping`. Бренд тепер може підтягуватися з будь-якого обраного атрибута WooCommerce, а не тільки з фіксованого поля.

**Інше**

Швидкий перехід до налаштувань плагіна зі сторінки "Плагіни" у WordPress. Версія плагіна оновлена до 2.1.0.
