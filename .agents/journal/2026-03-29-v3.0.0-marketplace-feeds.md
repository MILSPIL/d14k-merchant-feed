## Загальна суть змін

Плагін **d14k-merchant-feed** перетворився з генератора фідів виключно для Google Merchant Center на **універсальний генератор фідів** для GMC + українських маркетплейсів. Версія оновлена з `2.1.0` до `3.0.0`.

### Архітектурне рішення: білінгвальний single-feed

**Було (v2.x планувалось):** окремий YML-файл на кожну мову (`yml-horoshop-uk.xml`, `yml-horoshop-ru.xml`).

**Стало (v3.0):** один файл на канал із вбудованими перекладами:

- `<name>` = RU (базова мова YML-стандарту)
- `<name_ua>` = UA (переклад через WPML)
- `<description>` / `<description_ua>` — аналогічно

**Причина:** YML-стандарт (Prom.ua, Rozetka, Horoshop) вимагає RU як основну мову, а UA — як додатковий переклад. Нативно підтримується всіма трьома платформами. Два окремі файли створювали зайву складність і проблеми з маппінгом.

### 1. `includes/class-yml-generator.php` — **[НОВИЙ, 886 рядків]**

Центральний компонент v3.0. Повноцінний генератор YML XML-фідів для українських маркетплейсів.

**Клас:** `D14K_YML_Generator`
**Константа:** `CHANNELS = ['horoshop', 'prom', 'rozetka']`

**Ключові методи:**

| Метод | Призначення |
|-------|-------------|
| `generate($channel)` | Точка входу. Перемикає WPML на RU, будує XML, зберігає у `wp-content/uploads/d14k-feeds/yml-{channel}.xml` |
| `build_yml($channel)` | Збирає повний XML-документ: header → currencies → categories → offers |
| `build_offer(...)` | Генерує `<offer>` з platform-specific полями (7 параметрів) |
| `build_categories()` | Блок `<categories>` з ієрархією WooCommerce |
| `get_translated_value(...)` | Отримує UA-переклад через WPML (`wpml_object_id` filter) |
| `get_product_params(...)` | Витягує атрибути товару як `<param>` масив |
| `get_primary_category_id(...)` | Обирає найглибшу (найспецифічнішу) категорію |
| `format_price($price)` | Форматує ціну без валюти, крапка як роздільник |

**Platform-specific логіка в `build_offer()`:**

| Поле | Horoshop | Prom.ua | Rozetka |
|------|----------|---------|---------|
| `selling_type` | — | `"r"` (роздріб) | — |
| `group_id` | — | parent ID для варіацій | — |
| `in_stock` | — | available attr | — |
| `description` | HTML в CDATA | HTML в CDATA | text-only (strip_all_tags) |
| `oldprice` / `price_old` | `<oldprice>` | `<oldprice>` | `<price_old>` |
| `stock_quantity` | — | `<quantity_in_stock>` | `<stock_quantity>` (fallback: 1/0) |
| `article` / `vendorCode` | `<vendorCode>` | `<vendorCode>` | `<article>` |
| `country` | — | ISO→UA назва | — |
| `delivery` | `<delivery>true` | `<delivery>true` | — |
| `keywords` | — | product tags | — |
| `gtin` | — | meta `_gtin` | — |
| Max photos | 10 | 10 | 15 |

**Фільтрація** (перенесена з GMC генератора):

- Виключення за категоріями (з WPML-трансляціями)

### 2. `d14k-merchant-feed.php` — Головний файл плагіна

**Зміни:**

- **Версія:** `2.1.0` → `3.0.0`
- **Назва:** `MIL SPIL GMC Feed` → `MIL SPIL Feed Generator`
- **Опис:** Додано "та українських маркетплейсів (Horoshop, Prom.ua, Rozetka)"
- **Новий endpoint:** `/marketplace-feed/{channel}/` (rewrite rule)
- **Новий handler:** `d14k_yml_feed_handle_request()` — обробляє запити до YML-фідів
  - Перевіряє чи канал валідний (з CHANNELS константи)
  - Перевіряє чи канал увімкнений в налаштуваннях
  - Генерує фід on-the-fly якщо файл не існує
  - Content-Type: `application/xml`, Cache-Control: `max-age=3600`
- **Видалено** sub-daily cron інтервали (3h, 6h, 12h) — GMC забирає фід max раз на добу
- **Ін'єкція:** `D14K_YML_Generator` створюється і передається в `D14K_Cron_Manager` та `D14K_Admin_Settings`

### 5. `includes/class-wpml-handler.php` — Обробник WPML

**Зміни:**

- **Новий метод `get_default_language()`:**

  ```php
  public function get_default_language() {
      if (!$this->is_active()) {
          return substr(get_locale(), 0, 2);
      }
      return apply_filters('wpml_default_language', null);
  }
  ```

- Без цього методу виникав **fatal error** при генерації YML-фіду (використовується в `build_yml()` для отримання правильного home URL)

## Тестування фідів (порівняння з Horoshop export)

**Дата тестування:** 2026-03-29
**Повний звіт:** `.agents/references/feed-comparison-2026-03-29.md`

### Результати

- **110 із 112** товарів Horoshop знайдені в нашому фіді по vendorCode ✅
- **131/134** наших товарів мають `name_ua` (97.8%) ✅
- **22 товари Renew** — тільки в нашому фіді (не імпортовані в Horoshop)
- **Ціни** — свідомо різні (−1 грн на Horoshop), НЕ баг
- **Vendor = FILLER.COM.UA** замість реальних брендів — потрібно налаштувати `brand_mode=attribute` в адмінці

### Відкриті питання

1. Настроїти бренд з WooCommerce атрибуту (код готовий, тільки налаштування)
2. Додати `group_id` для Prom-каналу (потрібно для різновидів)
3. Імпортувати Renew в Horoshop або виключити з YML-фіду
