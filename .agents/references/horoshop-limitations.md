# Обмеження Horoshop — довідник для розробки

> Створено: 2026-03-31 | Статус: очікуємо відповідь підтримки Horoshop

## 1. Категорії НЕ створюються автоматично

**Проблема:** При імпорті CSV/XML Horoshop **не створює** категорії, яких ще немає в системі. Всі товари з неіснуючою категорією отримують помилку "Категорія не найдена" і пропускаються.

**Доказ:** Лог імпорту `log_import_31_03_2026.xlsx` — 134 товари, 100% помилок "Категорія не найдена".

**Поточний workaround:** Ручне створення категорій в адмінці Horoshop  
(*Товари → Категорії → Додати*) перед першим імпортом.

**Що було спробовано:**

- XML фід з тегом `<categories>` — ігнорується
- CSV з колонкою "Категорія" — ігнорується при першому імпорті
- API — доступний тільки на тарифі Pro (не у всіх клієнтів)

**Запит до підтримки:** Надіслано 2026-03-30, очікуємо відповідь.  
Питання: чи є інструмент масового створення категорій без API Pro?

---

## 2. WebP зображення не підтримуються

Horoshop не відображає WebP зображення в картках товарів.

**Рішення:** Увімкнути конвертацію WebP → JPG в налаштуваннях плагіна.  
Параметр: `webp_to_jpg` у `d14k_merchant_feed_settings`.

## 3. Маппінг полів CSV

Робочий порядок колонок для Horoshop CSV імпорту:

| # | Колонка | Примітки |
|---|---------|----------|
| 1 | Назва | `name_ua` з WooCommerce |
| 2 | Артикул | SKU |
| 3 | Ціна | Числовий формат |
| 4 | Стара ціна | Якщо є акція |
| 5 | Опис | `description_ua`, HTML |
| 6 | Категорія | Повний шлях: `Категорія > Підкатегорія` |
| 7 | Фото | URL через `;` (головне + галерея) |
| 8 | Наявність | `+` / `-` |

## 4. Prom.ua та Rozetka — статус тестування

| Платформа | Формат | Статус | Примітки |
|-----------|--------|--------|----------|
| Horoshop | CSV | ✅ Працює | Потребує ручних категорій |
| Prom.ua | YML | 🟡 Тестування | YML генератор готовий, не верифіковано |
| Rozetka | YML | 🔲 За розкладом | YML генератор готовий, не верифіковано |

---

## 5. Наступні кроки після відповіді підтримки

1. **Якщо є інструмент масового створення:**
   - Інтегрувати в плагін або задокументувати workflow
   - Оновити warning в UI

2. **Якщо API Pro — єдиний варіант:**
   - Додати опціональну інтеграцію з Horoshop API
   - Автоматичне створення категорій через API перед імпортом
   - Потрібен API ключ від клієнта (тариф Pro)

3. **Альтернатива:**
   - Генерувати CSV-файл ТІЛЬКИ з категоріями для попереднього імпорту
   - Або інструкцію по ручному створенню з CSV-переліком
